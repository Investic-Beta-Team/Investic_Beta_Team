Indicators' parameter optimisation¶

References

  • https://vectorbt.dev/api/indicators/factory/
  • https://vectorbt.dev/api/indicators/factory/#vectorbt.indicators.factory.IndicatorFactory.from_pandas_ta
  • https://vectorbt.dev/api/indicators/factory/#vectorbt.indicators.factory.IndicatorFactory.from_apply_func

Concept

  • การ Optimise Parameter ด้วย VectorBT จะต้องใช้ Indicator ที่ VectorBT รู้จักเท่านั้น ตอนนี้มี 3 รูปแบบก็คือ

  1. ใช้ Indicator ที่ติดมาจาก VectorBT เช่น vbt.MA() , vbt.MACD() แต่มีค่อนข้างน้อย คิดว่าน่าจะไม่เพียงพอต่อการใช้งานของทุกคน ดู Indicator ของ VectorBT อื่นๆ ได้ที่ https://vectorbt.dev/api/indicators/basic/

  2. แปลง Indicator ของ Pandas TA มาใช้งาน โดยใช้ IndicatorFactory.from_pandas_ta(indicator_name: str) โดยที่ indicator_name คือชื่อ Function ที่เรียกใช้งานใน Pandas TA เช่น ta.macd, ta.ema, ta.psar

  3. เขียน Indicator ขึ้นมาเอง ดูที่ https://vectorbt.dev/api/indicators/factory/#vectorbt.indicators.factory.IndicatorFactory

  • Indicator.run() จะเป็นการนำข้อมูลที่ Input ไปคำนวณตามสูตรของ Indicator เพียงครั้งเดียว แต่ parameters ของ indicator สามารถใส่เป็น list, tuple หรือ numpy.array ก็จะคำนวณ indication โดยใช้สามาชิกใน list และ tuple ได้

  • โดยปกติ Method run() ก็ใช้สำหรับคำนวณ Indicator อยู่แล้ว แต่ปกติเราจะกำหนด Parameter ของ Indicator เป็นเลขโดดๆ (int หรือ float) แต่ถ้าต้องการ optimise ก็สามารถใส่เป็น list หรือ numpy.array ก็ได้่ เช่น

แบบปกติ

macd_comb = vbtmacd.run(price_df['Close'], 
                        fast=14,
                        slow=26, 
                        signal=9,)

แบบต้องการ Optimise

macd_comb = vbtmacd.run(price_df['Close'], 
                        fast=np.arange(5, 50), 
                        slow=np.arange(5, 50), 
                        signal=9)
  • โดย default ของ VectorBT หากใส่ parameter เป็น list แล้ว จะต้องใช้จำนวนสมาชิกจำนวนเท่ากันทุกๆ parameter โดยตอนคำนวณ indicator จะเป็นการ map ไปยังตำแหน่งเดียวกัน เช่น หากต้องการหา MACD โดยกำหนด fast = [1, 2, 3, 4, 5] (จำนวน 5 ตัว) หากต้องการ slow = [6, 7, 8] (3 ตัว) จะทำไม่ได้ เพราะจำนวนสมาชิกไม่เท่ากัน จะต้องให้เป็น [6, 7, 8, 9, 10] ถึงจะทำได้

ดังนั้น Code สำหรับ Optimise ก็จะเป็น

macd_comb = vbtmacd.run(price_df['Close'], 
                        fast=np.arange(5, 50), 
                        slow=np.arange(5, 50), 
                        signal=9, 
                        param_product=True)

  • Indicator.run_comb() จะเป็นการรัน Indicator ชนิดเดียวกันหลายครั้ง (ตามจำนวน argument r ที่กำหนด - Default r=2) ให้เหมือนมีหลายๆ Indicator ร่วมกัน ในแต่ละ Indicator จะใช้ argument param_product=True เพื่อ permute ค่าของ param ภายใน indicator นั้นก็ได้

  • หากสร้าง Indicator ใหม่ (โดยใช้ class IndicatorFactory) ที่ทำการรวม Indicator พื้นฐานไว้แล้ว (เช่นใช้ MA Filter trend แล้วใช้ MACD บอกจุดเข้าออก) จะถือว่าเป็น 1 Indicator -> ให้ใช้ run() ในการ optimise ก็พอ

Key takeaways

  • หาก strategy ใช้ Indicator ชนิดเดียวกัน หลายชุด แต่มี parameter ที่ต่างกัน (เช่น MA 2 เส้นตัดกัน) ให้ใช้ run_comb() ในการ optimise เพราะว่าจะเป็นการคำนวณผลลัพธ์ของ 2 Indicator (MA ทั้ง 2 เส้น)

  • หาก strategy ใช้ Indicator เพียงชนิดเดียว (เช่น MACD ตัดกับ MACD Signal) ให้ใช้ run() ในการ optimise และใส่ argument param_product=True เพื่อทำการ permute ค่า parameter ของ indicator

  • หาก strategy ใช้ Indicator หลายชนิด ให้สร้าง Indicator ใหม่ที่รวม Indicator นั้น (ใช้ Class IndicatorFactory) แล้วใช้ run() ในการ optimise (เพราะเมื่อรวมกันแล้ว ก็เหมือนเป็น indicator ชนิดเดียว)

Multi-Indicators optimisation : EMA + MACD¶

เป็นตัวอย่างที่ไม่ดีเท่าไหร่ เพราะ Total return ลดลง แต่ก็ทำเป็นตัวอย่างให้่ดูว่าถ้าจะใช้ Indicator 2 ชนิดจะต้องทำยังไง

https://vectorbt.dev/api/indicators/factory/#vectorbt.indicators.factory.IndicatorFactory.from_apply_func

เริ่มต้นจากกำหนดการคำนวณต่างๆ ใน Indicator ก่อน ตัวอย่างนี้คือ

  • นำข้อมูลไปเข้า Indicator EMA และ MACD
  • คำนวณ Strategy คือ Entry เมื่อราคาปิดอยู่สูงกว่า EMA และ MACD ตัด Signal ขึ้น Exit คือราคาปิดต่ำกว่า EMA หรือ MACD ตัดลง

def custom_indicator(close, 
                     EMA_period = 50,
                     MACD_fast = 14,
                     MACD_slow = 26,
                     MACD_signal = 9) :

    vbtema = vbt.IndicatorFactory.from_pandas_ta('ema')
    vbtmacd = vbt.IndicatorFactory.from_pandas_ta('macd')
    
    """
    * output ของ .run() และเมื่อนำไปใช้งานอื่นๆ เช่นการ ใน custom indicator จะเป็น pd.DataFrame() ตอนนำไปสร้าง
    * strategy จะต้องทำให้เป็น series ก่อนด้วย .iloc[]
    """
    ema = vbtema.run(close, length=EMA_period)
    macd = vbtmacd.run(close,
                      fast=MACD_fast,
                      slow=MACD_slow,
                      signal=MACD_signal)
    
    entries = ema.ema_below(close).iloc[:, 0] & macd.macd_crossed_above(macd.macds).iloc[:, 0]
    exits = ema.ema_above(close).iloc[:, 0] | macd.macd_crossed_below(macd.macds).iloc[:, 0]
    
    return entries, exits
    # return result_df['entries'], result_df['exits']
